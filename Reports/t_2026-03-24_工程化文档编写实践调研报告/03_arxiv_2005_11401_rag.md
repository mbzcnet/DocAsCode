# 来源解读：arXiv 2005.11401｜Retrieval-Augmented Generation（RAG）

原始来源：
- https://arxiv.org/abs/2005.11401

延伸阅读（同论文的可读 HTML 版本）：
- https://ar5iv.labs.arxiv.org/html/2005.11401

## 论文要解决的核心矛盾

论文指出：大型预训练语言模型虽然在参数中“存储”了大量事实知识，但在知识密集型任务上仍存在关键局限：

- 不能轻易扩展/更新知识（更新成本高）
- 难以提供可追溯的“依据/来源”（provenance）
- 容易产生幻觉（hallucination）

因此作者提出：将“参数化记忆”（模型参数）与“非参数化记忆”（可检索的文档索引）结合，通过检索增强生成，让模型在生成时可以访问外部证据并提升可解释性与可更新性。

## RAG 的基本结构（从工程角度重述）

RAG = Retriever（检索器） + Generator（生成器）：

- Retriever：给定输入 x，返回与之相关的 Top-K 文档片段 z（论文使用 DPR 的双编码器结构 + MIPS 检索）
- Generator：在输入 x 的基础上拼接检索到的文档 z，生成输出 y（论文使用 BART-large，把 x 与 z 直接拼接）

论文将文档视为“潜变量”，在训练时对 Top-K 文档做边际化（marginalize），从而实现端到端的训练目标：最小化负对数似然。

## 两种 RAG 变体：RAG-Sequence vs RAG-Token

### RAG-Sequence（整段输出使用同一份证据）

- 假设：整段输出 y 由同一个检索到的文档 z 支撑
- 做法：对每个候选文档计算整段输出概率，再按检索先验加权求和
- 工程含义：更“文档级引用”，适合答案短、证据集中场景

### RAG-Token（每个 token 可来自不同证据）

- 假设：输出的不同 token 可以由不同文档支撑
- 做法：对每个输出位置 i，对 Top-K 文档边际化后得到该 token 的分布
- 工程含义：更“证据碎片级拼接”，适合需要整合多处资料的生成任务

## 论文的重要实验与结论（对落地最有用的点）

### 1) RAG 的优势不仅是“更准”，还是“更可控”

论文强调引入外部索引的好处包括：

- 生成更具体、更具事实性（对比纯 BART）
- 可通过替换索引来更新知识（index hot-swapping），无需重新训练模型
- 可一定程度缓解幻觉（因为生成更依赖显式文本证据）

### 2) “索引更新”是治理能力，而不仅是工程细节

论文展示了用不同年份的 Wikipedia 索引替换，模型回答“国家领导人是谁”会随索引变化而变化（匹配年份高，错配年份低），说明：

- 非参数化记忆是可热更新、可版本化的知识载体
- 这为工程系统提供了“版本语境”与“可审计数据来源”的抓手

### 3) 不同任务对检索策略的偏好不同

论文还做了对比：学习型稠密检索与 BM25 在不同任务上表现可能不同（例如实体强、词面匹配强的任务 BM25 可能更好）。对工程实践的启发是：

- 不要迷信单一检索范式
- 需要按任务做混合检索与重排策略的评测闭环

## 对“工程化文档编写 + 严谨引用”的直接启发

### A. 把“provenance”作为一等公民

论文点名“提供依据/来源”是纯参数模型的痛点之一。对应到文档工程，建议把“证据对象（evidence object）”纳入输出协议：

- 每个关键结论必须绑定证据片段（文档/段落/页码/版本/权限）
- 证据片段必须可追溯到具体来源（URL、仓库路径、commit/tag、时间戳）

### B. 用“索引版本”解决跨版本与上下文紧密的问题

你的场景资料多、上下文紧密，很容易出现“引用漂移”。RAG 的索引热替换思想可落到文档工程上：

- 索引与发布版本绑定（同一版本对应固定索引快照）
- 文档生成/改写必须指明使用的索引版本
- 审计时可复现：同样索引 + 同样提示词协议 → 可重跑核对

### C. 评测要同时覆盖检索与生成

论文做的不是单纯“生成效果”比较，而是包含检索质量、任务准确性等维度。对工程文档系统，建议评测维度至少包括：

- 检索相关性（取回的证据是否真的相关）
- 生成忠实度（回答是否严格基于证据）
- 引用覆盖率（关键结论是否都能回指证据）

## 建议你在本项目里怎么用（落地建议）

- 将 RAG 作为“证据供给机制”，而不是“增强聊天”的装饰功能
- 将索引构建视为“文档发布工艺”的一部分（版本化、可回滚、可审计）
- 将输出协议固定为“结论-依据-引用”，并在 CI/服务端做确定性校验（引用可解析、证据足够、无证据拒答）
